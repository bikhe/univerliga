
# FeedbackHub — Корпоративная система обратной связи

**FeedbackHub** — это платформа для сбора, анализа и управления обратной связью между сотрудниками. Система позволяет оценивать работу коллег в привязке к конкретным задачам, отслеживать динамику эффективности и формировать рейтинги на основе объективных данных.

## 2. Тестовые пользователи и Роли

Система содержит предварительно загруженные данные (пользователи, задачи, отзывы). Для входа используйте следующие учетные записи:

| Роль | Email | Пароль | Доступ и Возможности |
| :--- | :--- | :--- | :--- |
| **Руководитель** (Manager) | `manager@example.com` | `manager123` | **Полный доступ:** Аналитика, Отчеты, Управление задачами/тегами, Создание пользователей. |
| **Сотрудник** (Employee) | `employee1@example.com` | `employee123` | **Ограниченный доступ:** Только создание отзывов и просмотр своих входящих/исходящих. |
| **Администратор** (Admin) | `admin@example.com` | `admin123` | **Технический доступ:** Все права менеджера + доступ в Django Admin Panel. |

*Также доступны пользователи `employee2@...` до `employee5@...` с паролем `employee123`.*

## 3. Описание работы системы

### Архитектура
Проект построен на микросервисной архитектуре с использованием Docker Compose:
*   **Backend:** Python 3.11, Django 4.2, Django Rest Framework.
*   **Database:** PostgreSQL 15.
*   **Frontend:** React 18, TypeScript, Ant Design, Recharts.

### Основные сценарии

#### Процесс оценки (Feedback Loop)
1.  Пользователь выбирает коллегу из списка.
2.  Система подгружает активные задачи.
3.  Пользователь ставит оценку (**Рейтинг 1–5 звезд**).
4.  В зависимости от оценки (Позитивная >= 4 / Негативная <= 3) предлагаются соответствующие **теги** (компетенции).
5.  Пользователь пишет комментарий (минимум 10 символов) и отправляет отзыв.

#### Система ограничений (Anti-Spam)
Для обеспечения объективности на бэкенде действуют жесткие лимиты:
*   Макс. **5** отзывов в день.
*   Макс. **15** отзывов в неделю.
*   Макс. **2** отзыва на одного и того же коллегу в неделю.
*   Запрет на отзыв самому себе.
*   Уникальность: один отзыв на связку `[Автор -> Получатель -> Задача]`.

#### Аналитика и Отчеты (Для менеджера)
*   **Сводка:** Общее количество отзывов, NPS команды.
*   **Тренды:** График активности и тональности по дням/месяцам.
*   **Лидерборд:** Рейтинг сотрудников, сортируемый по **% Позитивности**. Формула: `(100 / (Pos + Neg)) * Pos`.
*   **Экспорт:** Генерация Excel-отчетов (`.xlsx`) за выбранный период (неделя, месяц, полгода).

## 4. Инструкция по локальному запуску

Если вы хотите запустить проект на своем компьютере.

### Предварительные требования
*   Установленный **Docker** и **Docker Compose**.
*   Свободные порты **3000** и **8001** (или измените их в конфиге).

### Шаг 1: Клонирование репозитория
```bash
git clone https://github.com/bikhe/univerliga.git
cd feedbackhub
```

### Шаг 2: Настройка окружения (Environment Variables)

В корне проекта находится файл `docker-compose.yml`. Он уже содержит базовые настройки.
Если вы запускаете проект локально, убедитесь, что переменные настроены на `localhost`.

**Ключевые переменные в `docker-compose.yml`:**

*   **Backend Service:**
    *   `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD` — доступы к БД.
    *   `ALLOWED_HOSTS` — разрешенные хосты (добавьте `localhost` и `127.0.0.1`).
    *   `CORS_ALLOWED_ORIGINS` — адрес фронтенда (обычно `http://localhost:3000`).

*   **Frontend Service:**
    *   `REACT_APP_API_URL` — **Самое важное!** Адрес, куда браузер будет слать запросы. Для локального запуска: `http://localhost:8001`.

### Шаг 3: Запуск контейнеров

```bash
docker-compose up --build -d
```
*Флаг `-d` запускает в фоновом режиме. Сборка может занять 2-5 минут.*

### Шаг 4: Инициализация Базы Данных

После запуска контейнеров база данных будет пустой. Необходимо применить миграции и загрузить демонстрационные данные.

1.  **Применение миграций** (создание таблиц):
    ```bash
    docker-compose exec backend python manage.py migrate
    ```

2.  **Загрузка демо-данных** (пользователи, задачи, отзывы):
    ```bash
    docker-compose exec backend python manage.py load_test_data
    ```
    *Эта команда автоматически создаст администратора, менеджера, сотрудников, набор задач и сгенерирует около 30 случайных отзывов для аналитики.*

### Шаг 5: Проверка работы

Откройте в браузере:
*   Frontend: **http://localhost:3000**
*   API: **http://localhost:8001/api/**
